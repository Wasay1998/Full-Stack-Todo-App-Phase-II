# Frontend System Skill

## Instructions

1. **Page Structure**
   - Define clear sections for each page (header, hero, content, footer)
   - Use semantic HTML tags (`<header>`, `<main>`, `<section>`, `<footer>`)
   - Maintain consistent layout across pages
   - Ensure responsive design for mobile and desktop

2. **Components**
   - Break UI into reusable components (buttons, cards, forms, modals)
   - Use props and state management where applicable
   - Keep components small, single-purpose, and composable
   - Follow accessibility best practices (ARIA roles, alt text)

3. **Layout**
   - Use CSS Flexbox or Grid for layout structure
   - Maintain spacing and alignment consistency
   - Support dynamic content resizing
   - Ensure cross-browser compatibility

4. **Styling**
   - Use a consistent color palette and typography
   - Apply hover, focus, and active states for interactive elements
   - Support themes (light/dark) if applicable
   - Optimize CSS for performance (avoid redundant styles)
   - Consider utility-first frameworks (Tailwind) or component libraries (Shadcn/UI)

5. **Animations & Interactivity**
   - Animate elements on load or scroll (fade-in, slide, scale)
   - Use smooth transitions for hover/focus interactions
   - Avoid excessive animations that hinder performance
   - Ensure accessibility (prefers-reduced-motio
